## simple RAG

- chunking
- chunks embedding

## semantic chunking
优化chunk的长度，不采用固定的大小，而是把语义接近的放到相同chunk
1. The text is divided into individual sentences.
2. Embeddings are created for each sentence.
3. The similarity between consecutive sentences is calculated.
4. "Breakpoints" are identified where the similarity drops significantly.
5. The sentences are grouped into chunks based on these breakpoints.

## Context-Enriched Retrieval
优化返回的chunk，除了返回K个最相似的chunk，把每个chunk相邻的M个chunk也一起返回

## Contextual Chunk Headers
chunk切分后，对每个chunk做个summry，然后和chunk正文一起做embedding和相似度召回

Benefits include:

- Best semantic match during search.
- Additional context for LLM to understand the chunk content.
- Clearer organization of retrieved information

## Document Enhancement
The process works like this:

1. For each chunk of text, generate several questions that could be answered with that chunk.
2. Create embeddings for both the original chunks and the generated questions.
3. Upon receiving a query from the user, search for matches in both chunks and questions.

生成fake question，作为一路单独的召回

## Query Transformation: Reformulating for Better Retrieval

Query transformation involves three main approaches:

1. **Rewriting the query**Rephrase the original query to make it clearer and more specific.
2. **Query “step-back”**Generate a broader query to obtain contextual information.
3. **Query decomposition**Split the query into smaller subqueries.

## Reranker: A Second Pass to Prioritize the Best
在召回K个最相近结果后，引入重排序，可以使用reranker模型也可以使用LLM判断

## Relevant Segment Extraction (RSE)

RSE的流程如下：

1. **文档切块**：还是老套路，先切成小块。
2. **相关性打分**：每块和用户问题算相关性分数。
3. **连续片段提取**：用“最大子段和”算法，找出一段段连续的高分块，拼成完整的上下文片段（设定一个window大小作为segment中包含的chunk数）。
4. **上下文重构**：把这些片段拼起来，作为大模型的输入。

给大模型的上下文更连贯

## Contextual Compression
对检索的上下文进行压缩，确保不超过上下文窗口大小，信息更密集相关

## Feedback Loop
检索并完成答案合成后，询问用户的反馈打分，用来优化后续

## Adaptive RAG

Adaptive RAC adapts the retrieval strategy to the specific type of question. It uses four different strategies:

- **Factual Strategy**For objective questions seeking specific facts.
- **Analytical Strategy**For questions that require analysis and synthesis of information.
- **Opinion Strategy**For questions seeking perspectives or opinions.
- **Contextual Strategy**For questions that require broad context.

## Self-RAG

Self-RAG is proactive and reflective, deciding whether to retrieve information, what to retrieve, and how to use it. It generates "reflection tokens" to justify its own process.

## Graph RAG

Graph RAC organizes information as a knowledge graph, where nodes represent concepts and edges represent the relationships between those nodes. This allows you to find information that is not only directly relevant, but also indirectly related.

## Hierarchical Indexes

This technique implements a two-stage retrieval process: first, it searches the abstracts to quickly identify the relevant sections, then it searches the detailed chunks within those sections.

## HyDE (Hypothetical Document Embedding)

HyDE generates a hypothetical document that answers the query, creates an embedding of that document, and then searches for documents similar to that embedding. The idea is that the hypothetical document can better capture the intent of the query.

Score: 0.5

## Fusion RAG

Fusion RAG combines different retrieval methods, such as vector (semantic) search and keyword search (BM25). This allows capturing both semantic meaning and exact matches.

Score: 0.83

## Multi-Modal RAG

Multi-Modal RAG uses textual and visual information (images, graphs) to improve responses. It extracts text and images, generates captions for the images, and creates embeddings for both.

Score: 0.79

## Corrective RAG (CRAG)

CRAG adds an evaluation step after the initial retrieval to verify the relevance of the retrieved documents. Depending on the relevance, it may proceed normally, use web search, or combine information from both sources.
