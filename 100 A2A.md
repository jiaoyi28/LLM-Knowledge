## 1. A2A 是什么

A2A（Agent2Agent）是一个面向 **Agent 与 Agent 之间通信协作** 的开放协议。它的目标不是规定 Agent 内部怎么实现，而是规定 **不同 Agent 如何发现彼此、交换消息、协同完成任务**。A2A 强调的是跨框架、跨语言、跨部署环境的互操作性。

从协议层看，A2A 支持几类核心能力：

- **发现（Discovery）**：客户端先找到目标 Agent 的 Agent Card
- **发送消息（Send Message）**：发起一次普通请求
- **流式消息（Send Streaming Message）**：边处理边返回事件
- **任务查询（Get Task）**：适合长任务、异步任务场景

## 1.1  HelloWorldAgent 示例
```python
from a2a.server.agent_execution import AgentExecutor, RequestContext
from a2a.server.events import EventQueue
from a2a.utils import new_agent_text_message


# --8<-- [start:HelloWorldAgent]
class HelloWorldAgent:
    """Hello World Agent."""

    async def invoke(self) -> str:
        return 'Hello World'


# --8<-- [end:HelloWorldAgent]


# --8<-- [start:HelloWorldAgentExecutor_init]
class HelloWorldAgentExecutor(AgentExecutor):
    """Test AgentProxy Implementation."""

    def __init__(self):
        self.agent = HelloWorldAgent()

    # --8<-- [end:HelloWorldAgentExecutor_init]
    # --8<-- [start:HelloWorldAgentExecutor_execute]
    async def execute(
        self,
        context: RequestContext,
        event_queue: EventQueue,
    ) -> None:
        result = await self.agent.invoke()
        await event_queue.enqueue_event(new_agent_text_message(result))

    # --8<-- [end:HelloWorldAgentExecutor_execute]

    # --8<-- [start:HelloWorldAgentExecutor_cancel]
    async def cancel(
        self, context: RequestContext, event_queue: EventQueue
    ) -> None:
        raise Exception('cancel not supported')

    # --8<-- [end:HelloWorldAgentExecutor_cancel]
```

## 2. HelloWorldAgent 示例的本质

你前面那段代码本质上分成两层：

### 2.1 Agent 逻辑层
```python
class HelloWorldAgent:  
    async def invoke(self) -> str:  
        return 'Hello World'
```

这一层只关心业务逻辑。  
在这个示例里，它没有读取输入，也没有调用模型，只是固定返回 `"Hello World"`。

### 2.2 A2A 执行器层

```python
class HelloWorldAgentExecutor(AgentExecutor):  
    async def execute(self, context: RequestContext, event_queue: EventQueue) -> None:  
        result = await self.agent.invoke()  
        await event_queue.enqueue_event(new_agent_text_message(result))
```

这一层负责和 A2A 框架对接：

- 接收请求上下文 `RequestContext`
- 调用实际 Agent
- 把结果通过 `EventQueue` 返回给调用方

所以，`HelloWorldAgent` 是“干活的人”，`HelloWorldAgentExecutor` 是“把它接入 A2A 协议的人”。这正是 A2A Python quickstart 里 `Agent Executor` 一节演示的模式。

---

## 3. `execute()` 到底做了什么

在 A2A 中，`execute()` 是服务端执行入口之一。它的职责不是监听端口，也不是决定部署地址，而是：

1. 从请求里拿上下文
2. 执行业务逻辑
3. 往事件队列里写结果

在 HelloWorld 示例里，流程是：

1. `self.agent.invoke()` 返回 `"Hello World"`
2. `new_agent_text_message(result)` 把结果包装成一条文本消息
3. `event_queue.enqueue_event(...)` 把消息发回调用方

因此，这个例子属于 **简单直接返回 Message** 的模式，而不是长时间运行的 Task 模式。A2A 规范明确说明：`message/send` 的结果可以是一个直接 `Message`，也可以是一个 `Task`。

---

## 4. Message 和 Task 的区别

A2A 里最容易混淆的是 `Message` 和 `Task`。

### 4.1 Message

适合：

- 简单问答
- 很快能返回结果
- 不需要状态追踪

HelloWorld 示例就是这种模式。A2A 规范允许服务端对简单交互直接返回 `Message`。

### 4.2 Task

适合：

- 长时间运行的处理
- 需要状态更新
- 需要支持中间结果、流式更新、用户补充输入

A2A 文档中 `Task` 是协议里的核心动作单元，具有状态、历史、产物等信息。状态包括 `submitted`、`working`、`completed`、`failed`、`canceled`、`input-required`、`rejected`、`auth-required` 等。

---

## 5. Agent 之间不是直接 import 调用

这是理解 A2A 的关键点。

### 5.0 调用示例代码

```python
import logging

from typing import Any
from uuid import uuid4

import httpx

from a2a.client import A2ACardResolver, A2AClient
from a2a.types import (
    AgentCard,
    MessageSendParams,
    SendMessageRequest,
    SendStreamingMessageRequest,
)
from a2a.utils.constants import (
    AGENT_CARD_WELL_KNOWN_PATH,
    EXTENDED_AGENT_CARD_PATH,
)


async def main() -> None:
    # Configure logging to show INFO level messages
    logging.basicConfig(level=logging.INFO)
    logger = logging.getLogger(__name__)  # Get a logger instance

    # --8<-- [start:A2ACardResolver]

    base_url = 'http://localhost:9999'

    async with httpx.AsyncClient() as httpx_client:
        # Initialize A2ACardResolver
        resolver = A2ACardResolver(
            httpx_client=httpx_client,
            base_url=base_url,
            # agent_card_path uses default, extended_agent_card_path also uses default
        )
        # --8<-- [end:A2ACardResolver]

        # Fetch Public Agent Card and Initialize Client
        final_agent_card_to_use: AgentCard | None = None

        try:
            logger.info(
                f'Attempting to fetch public agent card from: {base_url}{AGENT_CARD_WELL_KNOWN_PATH}'
            )
            _public_card = (
                await resolver.get_agent_card()
            )  # Fetches from default public path
            logger.info('Successfully fetched public agent card:')
            logger.info(
                _public_card.model_dump_json(indent=2, exclude_none=True)
            )
            final_agent_card_to_use = _public_card
            logger.info(
                '\nUsing PUBLIC agent card for client initialization (default).'
            )

            if _public_card.supports_authenticated_extended_card:
                try:
                    logger.info(
                        f'\nPublic card supports authenticated extended card. Attempting to fetch from: {base_url}{EXTENDED_AGENT_CARD_PATH}'
                    )
                    auth_headers_dict = {
                        'Authorization': 'Bearer dummy-token-for-extended-card'
                    }
                    _extended_card = await resolver.get_agent_card(
                        relative_card_path=EXTENDED_AGENT_CARD_PATH,
                        http_kwargs={'headers': auth_headers_dict},
                    )
                    logger.info(
                        'Successfully fetched authenticated extended agent card:'
                    )
                    logger.info(
                        _extended_card.model_dump_json(
                            indent=2, exclude_none=True
                        )
                    )
                    final_agent_card_to_use = (
                        _extended_card  # Update to use the extended card
                    )
                    logger.info(
                        '\nUsing AUTHENTICATED EXTENDED agent card for client initialization.'
                    )
                except Exception as e_extended:
                    logger.warning(
                        f'Failed to fetch extended agent card: {e_extended}. Will proceed with public card.',
                        exc_info=True,
                    )
            elif (
                _public_card
            ):  # supports_authenticated_extended_card is False or None
                logger.info(
                    '\nPublic card does not indicate support for an extended card. Using public card.'
                )

        except Exception as e:
            logger.error(
                f'Critical error fetching public agent card: {e}', exc_info=True
            )
            raise RuntimeError(
                'Failed to fetch the public agent card. Cannot continue.'
            ) from e

        # --8<-- [start:send_message]
        client = A2AClient(
            httpx_client=httpx_client, agent_card=final_agent_card_to_use
        )
        logger.info('A2AClient initialized.')

        send_message_payload: dict[str, Any] = {
            'message': {
                'role': 'user',
                'parts': [
                    {'kind': 'text', 'text': 'how much is 10 USD in INR?'}
                ],
                'messageId': uuid4().hex,
            },
        }
        request = SendMessageRequest(
            id=str(uuid4()), params=MessageSendParams(**send_message_payload)
        )

        response = await client.send_message(request)
        print(response.model_dump(mode='json', exclude_none=True))
        # --8<-- [end:send_message]

        # --8<-- [start:send_message_streaming]

        streaming_request = SendStreamingMessageRequest(
            id=str(uuid4()), params=MessageSendParams(**send_message_payload)
        )

        stream_response = client.send_message_streaming(streaming_request)

        async for chunk in stream_response:
            print(chunk.model_dump(mode='json', exclude_none=True))
        # --8<-- [end:send_message_streaming]


if __name__ == '__main__':
    import asyncio

    asyncio.run(main())
```

### 5.1 直接 import 调用

例如：

```python
agent = HelloWorldAgent()  
result = await agent.invoke()
```

这是**本地函数调用**，不属于 A2A 协议调用。

### 5.2 A2A 调用

A2A 的正确调用方式是：

1. 先使用`A2ACardResolver`找到目标 Agent 的 **Agent Card**
2. 用 `A2AClient` 创建客户端
3. 发送 `message/send` 或 `message/stream`
4. 目标 Agent 的 `execute()` 被触发
5. 结果通过消息或任务形式返回

也就是说，其他 Agent 调用你的 HelloWorldAgent 时，应该把它当成一个 **远程 A2A 服务**，而不是把类直接 import 进去。A2A 协议和 quickstart 都是按这个方式设计的。

---

## 6. Agent Card 是什么

A2A 中每个服务都需要有一个 **Agent Card**。  
它是一个 JSON 文档，用来描述这个 Agent 的：

- 身份信息
- 能力与技能
- 服务地址
- 身份认证要求
- 交互方式

规范明确要求 A2A Server 提供 Agent Card，客户端依赖它来发现和配置与目标 Agent 的交互。

可以把 Agent Card 理解成：

> “这个 Agent 是谁、会做什么、在哪里、怎么调用它”的说明书。

---

## 7. `base_url` 是什么

`base_url` 不是协议固定值，也不是 `HelloWorldAgentExecutor` 里定义的变量。  
它表示的是 **某个 A2A 服务对外暴露的根地址**。客户端通常基于它去获取 Agent Card，并继续发消息。官方 Python quickstart 的交互示例里使用了：

base_url = "http://localhost:9999"

这是一个示例地址，不是协议默认值。

---

## 8. 为什么在 `HelloWorldAgentExecutor` 里找不到 `base_url`

因为地址不属于执行器负责的内容。

### 8.1 执行器负责什么

执行器只负责：

- 收到请求后执行逻辑
- 产生消息 / 任务结果

### 8.2 服务启动层负责什么

真正决定地址的是 **服务启动层**，例如：

- 监听哪个 host / port
- 对外发布哪个 Agent Card
- 将 HTTP 请求路由给执行器

因此，`base_url` 通常出现在：

- `__main__.py`
- `server.py`
- `app.py`
- 部署配置（如 Docker、Ingress、Nginx）

而不是 `HelloWorldAgentExecutor` 里。A2A quickstart 文档中，Agent Card 和 server build / 启动是分开的章节，也体现了这个分层。

---

## 9. 多个 A2A Agent 部署时，`base_url` 是相同还是不同

**不一定，但通常应该不同。**

### 9.1 常见方式一：不同端口

例如：

- `http://localhost:9999`
- `http://localhost:10000`
- `http://localhost:10001`

此时每个 Agent 都有独立 `base_url`。

### 9.2 常见方式二：同域名，不同路径前缀

例如：

- `https://example.com/agent1`
- `https://example.com/agent2`
- `https://example.com/agent3`

此时域名相同，但每个 Agent 的根路径不同，因此严格说 `base_url` 也不同。

### 9.3 为什么通常不能完全共用同一个 `base_url`

因为客户端往往会基于某个 `base_url` 去定位并获取 Agent Card。一个服务入口通常对应一个 Agent Card；如果多个完全独立的 Agent 共用完全相同的入口，发现机制就会变得不明确。A2A 规范对 Agent Card 的发现有明确要求，这也是为什么实际部署时通常要做到“一个 Agent 服务，一个清晰可发现的入口”。

---

## 10. 一个典型的 A2A 调用链路

结合 HelloWorld，可以把调用链总结成下面这样：

1. **HelloWorld 服务启动**
2. 对外暴露 Agent Card
3. 调用方 Agent 使用 `base_url` 找到它的 Agent Card
4. 调用方创建 `A2AClient`
5. 调用方发送 `message/send` 或 `message/stream`
6. 目标服务路由到 `HelloWorldAgentExecutor.execute()`
7. `execute()` 调用 `HelloWorldAgent.invoke()`
8. 返回一条 `Message` 或一个 `Task`

这个过程正是 A2A 所强调的“Agent 之间通过协议协作，而不是共享内部实现”。

---

## 11. HelloWorld 示例的局限性

这个示例很适合入门，但它有明显限制：

### 11.1 不接收输入

`invoke()` 不带参数，所以无论外部传什么，它都返回同一个结果。

### 11.2 不支持复杂状态

它直接返回文本消息，没有任务状态流转。

### 11.3 不支持取消

如果 `cancel()` 只是抛异常，那么长任务场景下就不够用了。

### 11.4 没有错误处理

真实服务通常需要处理异常、鉴权、重试、输入校验等。

---

## 12. 从 HelloWorld 走向真实 Agent 的演进方向

一个更实用的 A2A Agent，一般会逐步增加下面这些能力：

- 从 `RequestContext` 中读取用户输入
- 将输入传给真正的业务逻辑或大模型
- 返回结构化输出或文件
- 用 `Task` 表示长时间任务
- 通过流式事件反馈中间进度
- 支持取消、失败、输入补充、鉴权等状态

这也是 A2A 规范中 `Message`、`Task`、状态流和发现机制存在的意义。

## 13. 常见误区

### 误区 1：A2A 就是把别的 Agent 的 Python 类 import 进来

不是。那只是本地代码复用，不是 A2A 协议通信。A2A 关注的是标准化的 Agent 间交互。

### 误区 2：`localhost:9999` 是 A2A 默认地址

不是。那只是官方 quickstart 的示例地址。

### 误区 3：`base_url` 应该写在 `AgentExecutor` 里

不是。地址通常由服务启动层和部署层负责。

### 误区 4：所有 Agent 都必须走 Task

不是。简单场景可以直接返回 `Message`。

---

## 14. 一段最简结论

A2A 的核心思想可以概括为一句话：

> **Agent 的内部实现可以完全不同，但它们可以通过统一协议发现彼此、发送消息、跟踪任务，并协同完成工作。**

你前面的 HelloWorld 示例展示的是最小化服务端实现：  
它通过 `AgentExecutor` 接收请求，通过 `EventQueue` 返回结果；而真正的服务地址、Agent Card 暴露和外部调用入口，属于服务启动层与部署层，而不是 Agent 业务类本身。
